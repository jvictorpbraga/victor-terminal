# claude-terminal — Specification

**Owner:** Victor
**Created:** 2026-05-06
**Status:** Spec locked, Phase 0 in progress

---

## What it is

A Windows-native terminal application that replaces `cmd.exe` for the user's primary use case: running Claude Code with full machine access. Built with **Tauri 2 + xterm.js + Claude Agent SDK + SQLite**.

It is a *terminal first*, *Claude integration second* — not an IDE, not a chat app. The aesthetic and interaction model is "command prompt that does not suck", not "Cursor without files".

---

## Why it exists (the problem)

cmd.exe (and even Windows Terminal in this user's experience) has friction the user hits daily while running Claude Code:

1. Copying/pasting large blocks of text breaks (line endings, character mangling, truncation).
2. Ctrl+A select-all is broken or absent.
3. Visual presentation is dated — no GPU rendering, no anti-aliasing, no modern font handling.
4. No durable record of what Claude did across compactions and restarts. When a Claude session compacts, useful detail is lost. When it ends, history scatters across multiple `.jsonl` transcript files no human reads.
5. No graceful recovery when Claude's context fills — the user has to manually start over.

---

## Non-goals (what this is NOT)

- Not a multi-tab terminal emulator with profiles, SSH, etc. (One window, one shell, one Claude.)
- Not a code editor. Optional code-viewer panel may come later, but v1 is terminal + Claude.
- Not cross-platform. Windows-only for v1. macOS/Linux later if it earns the right.
- Not a Claude wrapper that hides the terminal. The terminal IS the primary surface.

## Functional requirements

### F1 — Terminal core
- xterm.js renderer (GPU-accelerated via WebGL addon).
- PTY backend connected to PowerShell as default host shell, with full inherited permissions.
- Copy/paste must handle: 100k+ character pastes, mixed line endings (CRLF/LF), Unicode, ANSI escape sequences, file-path drag/drop.
- Ctrl+A = select all visible terminal output.
- Ctrl+C / Ctrl+V = copy / paste (with smart distinction from Ctrl+C-as-SIGINT when no selection).
- Mouse wheel + scrollback = native browser-feel scroll, 10k+ line buffer.
- Resize = correct PTY resize (no garbled output).

### F2 — Claude integration
- @anthropic-ai/claude-agent-sdk runs in a Node sidecar Tauri spawns.
- Default mode: **bypassPermissions** (full machine access — same surface Claude has in cmd today).
- API key stored via Tauri's secure storage (Windows Credential Manager backend).
- Streaming responses pipe directly into xterm.js with proper rendering.
- All built-in Claude tools enabled (Bash, Read, Write, Edit, Glob, Grep, Agent, etc.).

### F3 — Session Ledger (the durable memory)
- For each Claude tool call, an entry is appended to `sessions/{session-id}.md`:
  ```
  ## 2026-05-06 14:32:11 — Bash
  Started backend dev server on port 3001.
  Result: server up, no errors.
  ```
- Entries are *human-summary-style*, not raw tool args. Generated by a small post-tool hook that asks Claude (or a heuristic) to describe what just happened in 1-2 lines.
- Ledger is also indexed in SQLite for search.
- Ledger file is the user-facing source of truth: openable in any markdown reader.

### F4 — Auto-resume on context full
- Token counter tracks current Claude session usage continuously.
- At ~95% of context window, trigger fires:
  1. Final ledger entry written ("Session approaching context limit, resuming...")
  2. Current SDK session ends gracefully
  3. New SDK session starts in the same Tauri tab — terminal NOT cleared, user input area preserved
  4. New session's first system message includes: (a) Anthropic's compact summary + (b) the full ledger contents
  5. UI shows a small `↻ resumed` marker inline in the scrollback
- User experience: typing continues seamlessly, no lost work, no restart prompt.

### F5 — UI
- Modern minimal aesthetic. Reference: Warp/Wave but stripped down.
- Dark mode default, single accent color, generous whitespace.
- One main view: the terminal. Optional collapsible left sidebar listing past sessions (each = one ledger file).
- No toolbar, no menu bar (Tauri custom titlebar with just minimize/maximize/close).
- Custom mono font (JetBrains Mono or similar).

## Non-functional requirements

- **Binary size:** under 30 MB installed (Tauri target).
- **Cold start:** under 1 second to interactive terminal.
- **Memory:** under 200 MB idle (well below Electron equivalents).
- **All data local** — no telemetry, no cloud sync (yet). Ledger and SQLite live in `%APPDATA%\claude-terminal\`.
- **Anthropic API key** is the only external credential.

---

## Acceptance criteria (v1 ships when)

1. App launches, opens a real PowerShell session, accepts commands like cmd.exe does today — including all PATH inheritance.
2. Pasting a 50KB code block works without character loss or visual breakage.
3. Ctrl+A selects all visible terminal text.
4. Typing `claude` (or built-in command) starts an embedded Claude session that has Bash/Read/Write/Edit and can edit files anywhere on the machine.
5. After running Claude tools for ~5 minutes, the session-N.md ledger contains readable, descriptive entries for each action.
6. Forcing a context-fill scenario triggers seamless resume: user does not see the session restart, new Claude reads the ledger and continues coherently.
7. Closing and reopening the app preserves session history in the sidebar.

---

## Risks

- **Tauri + Node sidecar complexity** — IPC between Rust backend, Node SDK process, and xterm.js renderer is the most failure-prone wiring. Mitigated by Phase 1 spike before features.
- **node-pty Windows builds** — historically painful. Researcher is investigating Rust `portable-pty` as alternative.
- **Auto-resume coherence** — Claude may behave incoherently after resume even with ledger. Phase 4 needs a real torture test.
- **Rust toolchain install** — 5GB+ install for the user. Decision pending before Phase 1.

---

## Decisions log

- **2026-05-06** — Picked Tauri over Electron. Reason: user's stated priority is "more optimised, smoother, less bloat" — Tauri's 10-15 MB native binary serves that better than Electron's 150+ MB. Cost: requires Rust install.
- **2026-05-06** — Single-window, single-shell, single-Claude. Reason: explicit user statement "just clean terminal with history" — no multi-tab, no SSH, no IDE features in v1.
