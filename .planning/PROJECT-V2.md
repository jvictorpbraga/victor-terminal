# Victor Terminal — V2 Vision (2026-05-07)

> The pivot: stop being a terminal wrapper, become a native Claude chat app.
> Same engine (claude-code CLI), totally different UX.

---

## What this becomes

A **native Windows app** that *uses* claude-code CLI as its backend but renders everything in React — chat bubbles, sidebar, attachments, model picker. Like the web version of Claude (claude.ai), but local, snappy, with full claude-code tool-use power and Victor Braga's branding.

**Branding (DONE in Phase 1):**
- Renamed `claude-terminal` → `Victor Terminal` (`tauri.conf.json` productName, window title, App.tsx titlebar text, `index.html` title)
- Identifier kept as `app.claude.terminal` (changing it breaks the Windows installer upgrade path; not user-visible)
- New metallic "Vt" italic icon — silver gradient on dark navy, diagonal sheen, generated by `scripts/make-icon.ps1` then expanded by `tauri icon` to all formats (.ico for Windows, .icns for macOS, all PNG sizes, iOS/Android variants)
- Bottom-right silver italic "Victor Braga" signature — Playfair Display font, silver gradient via `-webkit-background-clip: text`

## Phase 2 — Chat UI rewrite (THE BIG ONE)

**Drop xterm entirely.** Replace it with a React chat surface. The bottom of the window is a prompt input, the rest is the message history.

### Backend: drive claude-code via stream-json

Instead of `claude --dangerously-skip-permissions` running as a TUI, spawn:

```
claude --print
       --input-format=stream-json
       --output-format=stream-json
       --include-partial-messages
       --include-hook-events
       --dangerously-skip-permissions
       [--continue OR --resume <id> for sidebar clicks]
       [--model <slug> if user picked one]
```

Then:
- Write a JSON line to claude's stdin per user message: `{"type":"user","message":{"role":"user","content":[{"type":"text","text":"..."}]}}`
- Read claude's stdout line-by-line — JSON events for tool_use, tool_result, text deltas, thinking deltas, usage updates
- Feed each event into a React store; the UI re-renders as a streaming chat

**Why this works:** claude-code already supports this exact flow for SDK integration. We just become a UI for that protocol. The session JSONLs we already track (`~/.claude/projects/<cwd>/<id>.jsonl`) are the same ones claude writes in --print mode, so the existing Ledger code keeps working.

**Implementation files (new):**
- `src-tauri/src/claude_session.rs` — manages a child claude-code process, owns its stdin/stdout, broadcasts JSON events to the frontend via `claude-event` Tauri event
- `src/state/conversation.ts` — React store (zustand or context) holding the current message list, streaming state, tool-call expansion state
- `src/components/Chat.tsx` — message list, renders user / assistant bubbles
- `src/components/Message.tsx` — single message bubble with role styling, copy button, expandable code/thinking blocks
- `src/components/PromptBar.tsx` — bottom input with `+` attach button, model picker, send button, paste handler for images
- `src/components/Welcome.tsx` — "How can I help you today?" centered hero before any message exists

### Components

**Welcome.tsx** — center of screen when no messages yet. Big "How can I help you today?" in the same Playfair italic feel as the signature. Disappears once a message is sent.

**Chat.tsx** — vertical scrolling list of messages. Auto-scrolls to bottom on new content unless user has scrolled up.

**Message.tsx** — bubble with:
- Role-styled background (user = subtle blue tint, assistant = neutral)
- Markdown-rendered text (use `react-markdown` + `remark-gfm`)
- Code blocks with syntax highlighting (use `shiki` for VS Code-quality)
- Tool calls: collapsed by default, click to expand and see input/output
- Thinking blocks: collapsed by default, hidden behind a "💭 Thinking" disclosure
- Copy button (top-right of bubble) → copies the rendered text content

**PromptBar.tsx** — sticky bottom bar:
- `+` button → opens file picker for images/files. Selected files render as chips above the textarea
- Textarea (auto-grow up to ~10 lines, then scroll). Enter = send, Shift+Enter = newline
- Image paste from clipboard: Ctrl+V with an image in clipboard → save to temp file, attach as image
- Model picker: dropdown of available claude models (opus / sonnet / haiku, with version slugs from claude-code)
- Send button (↑) — disabled while streaming

### Streaming/rendering strategy

When a user message is sent:
1. Push user message into store
2. Send JSON line to claude stdin
3. As stream events arrive:
   - `message_start` → create new assistant message with empty content
   - `content_block_delta` (text) → append text delta to current block
   - `content_block_start` (thinking) → create thinking block
   - `content_block_delta` (thinking) → append delta
   - `content_block_start` (tool_use) → create tool call block
   - `content_block_delta` (input_json_delta) → accumulate tool input
   - `tool_result` (from user-role event) → match to its tool_use_id, attach result
   - `message_stop` → mark message complete, re-enable send

## Phase 3 — Conversations sidebar (replaces Ledger panel)

Replace right-side modal Ledger with **left-side dropdown panel**. Click the menu icon (top-left, replaces "≡") → panel slides in from the left.

**File:** `src/components/Sidebar.tsx`

- Lists past conversations from `~/.claude/projects/<cwd>/*.jsonl`, sorted by mtime
- Each row: title (auto-generated `ai-title` if present, else first user message), date, token count
- Click row → app loads that conversation:
  - Spawn fresh claude-code with `--resume <session-id>`
  - Frontend reads the existing JSONL and replays events into the chat store
  - User can continue chatting, claude resumes where it left off
- Top of sidebar: `+ New chat` button — kills current claude, spawns fresh

## Phase 4 — Endless context (background restart)

**Key UX promise:** the user never sees their conversation reset. Even when context fills, claude transparently restarts in the background and keeps going.

### Mechanism

- Drop trigger from 92% to **75%** (`RESUME_THRESHOLD_PCT` in App.tsx)
- When idle ≥5s above threshold:
  1. Spawn a NEW claude-code child process in the background with `--continue` and a `--append-system-prompt-file` pointing at the running ledger MD
  2. Wait for it to be ready
  3. Atomically swap the active session: route subsequent stdin to the new child, drain stdout from the old one, kill the old one
  4. The frontend's conversation store keeps the same logical session — UI shows a single uninterrupted chat
- The **sidebar** keeps showing this as ONE conversation (we track our own "logical session ID" that wraps multiple underlying claude-code session IDs). New child sessions only show up internally, not in the sidebar list

### Action ledger (the "100% context" handoff)

Current `ledger_watcher.rs` writes a high-level transcript. Beef it up to capture EVERYTHING:
- Every Bash command + exit code
- Every file read with line ranges
- Every file edit with diff summary
- Every web fetch with URL
- Every thinking block (compressed)
- Every tool result truncated to ~200 chars

Format the MD as a *recipe* the next claude can follow: "I was working on X. I have already done A, B, C (with relevant file:line cites). Continue from here."

When the auto-resume fires, it points the new claude at this MD — claude reads it via the Read tool when needed (or the system prompt can include the last 8KB inline so context is immediate).

## Phase 5 — Attachments

- Ctrl+V image paste: `navigator.clipboard.read()` → check for image MIME → write to `%TEMP%\victor-terminal\paste-<uuid>.png` → attach to next message via claude-code's image input format (base64 in content array)
- `+` button: native file picker via `@tauri-apps/plugin-dialog`, multiple files, accepts images and any file type
- Selected files render as chips above the textarea, x to remove
- On send: build the user message with `content: [{type:"text",...}, {type:"image",source:{...}}]`

### claude-code CLI flags we depend on (from `claude --help`)

- `--print` — non-interactive mode
- `--input-format stream-json` — JSON line per message via stdin
- `--output-format stream-json` — JSON line per event via stdout
- `--include-partial-messages` — get streaming deltas
- `--include-hook-events` — get tool lifecycle events
- `--dangerously-skip-permissions` — same as today, no permission prompts
- `--continue` — pick up the most recent session in cwd
- `-r, --resume <id>` — resume specific session
- `--model <slug>` — pick model
- `--append-system-prompt <text>` — extra system prompt for context handoff
- `--session-id <uuid>` — set session ID explicitly (useful for our logical-session wrapping)
